<p>You can learn more about Hyperledger Fabric, including how to set up your local development environment, by following the Hyperledger Fabric tutorials available here: <a href="http://hyperledger-fabric.readthedocs.io/en/release-1.1/tutorials.html" rel="nofollow">http://hyperledger-fabric.readthedocs.io/en/release-1.1/tutorials.html</a></p>

<p>Hyperledger Composer is a framework or layer that builds on top of Hyperledger Fabric, that provides functionality that allows you to model your blockchain business network, and easily generate RESTful APIs and end user applications.</p>
<p>You can learn more about Hyperledger Composer, including how to set up your local development environment, by following the Hyperledger Composer tutorial available here: <a href="https://hyperledger.github.io/composer/latest/tutorials/tutorials.html" rel="nofollow">https://hyperledger.github.io/composer/latest/tutorials/tutorials.html</a></p>

<p>Navigate to the DevOps toolchain page, and click on the "Delivery Pipeline" button. You should see the following page, giving you an overview of the current status of your delivery pipeline:</p>

<p>The delivery pipeline is made up of two phases, "BUILD" and "DEPLOY".</p>
<p>The "BUILD" phase of the delivery pipeline clones your GitHub repository, installs any dependencies, and runs all of the automated unit tests for all of your smart contracts. If any unit tests fail, then the delivery pipeline will fail and your changes will not be deployed.</p>
<p>The "DEPLOY" phase of the delivery pipeline deploys your smart contracts into the IBM Cloud. It is reponsible for provisioning and configuring an instance of the IBM Blockchain Platform: Starter Plan (the blockchain network), an instance of Cloudant (the wallet for blockchain credentials), deploying the smart contracts, and deploying RESTful API servers for each deployed smart contract.</p>
<p>If you click "View logs and history", you can see the latest logs for your build:</p>
<p><a target="_blank" rel="noopener noreferrer" href="/vkozeny/vk-blockchain-starter-kit/blob/master/media/delivery-pipeline-logs.png"><img src="/vkozeny/vk-blockchain-starter-kit/raw/master/media/delivery-pipeline-logs.png" alt="Delivery Pipeline logs" style="max-width:100%;"></a></p>
<p>Both "BUILD" and "DELIVERY" phases should be green and showing that no errors have occurred. If this is not the case, you must use the logs to investigate the cause of the errors.</p>

<p>Currently, this starter kit does not assist you in creating applications that can interact with smart contracts developed using Hyperledger Fabric. You will need to develop your own applications from scratch.</p>
